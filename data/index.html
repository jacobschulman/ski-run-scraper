Available Endpoints

Latest Data (All Resorts)

GET /data/latest.json

Returns the most recent data for all configured resorts in a single JSON file.

https://{username}.github.io/{repo}/data/latest.json

Data Index

GET /data/index.json

Returns a manifest of all available data files for all resorts, including file counts and latest dates.

https://{username}.github.io/{repo}/data/index.json

Resort-Specific Data (by date)

GET /data/{resort}/{YYYY-MM-DD}.json

Returns data for a specific resort on a specific date.

https://{username}.github.io/{repo}/data/keystone/2025-01-05.json
https://{username}.github.io/{repo}/data/vail/2025-01-05.json

Data Structure

Each resort's data file contains:

  • Date: ISO 8601 timestamp
  • ResortId: Numeric resort identifier
  • GroomingAreas: Array of areas with trail information
    • Trail name, difficulty, open status, groomed status
    • Trail length, type, and other metadata
  • Lifts: Array of lift information (status, wait times, etc.)

Example Usage

JavaScript / Node.js

// Fetch latest data for all resorts
const response = await fetch('https://{username}.github.io/{repo}/data/latest.json');
const data = await response.json();

// Access specific resort
const keystoneData = data.keystone.data;
console.log(`Keystone has ${keystoneData.GroomingAreas.length} areas`);

Python

import requests

# Fetch specific resort data
url = 'https://{username}.github.io/{repo}/data/keystone/2025-01-05.json'
response = requests.get(url)
data = response.json()

print(f"Resort ID: {data['ResortId']}")
print(f"Total areas: {len(data['GroomingAreas'])}")

LED Matrix / IoT

// Fetch only the data you need for a specific resort
fetch('https://{username}.github.io/{repo}/data/vail/2025-01-05.json')
  .then(res => res.json())
  .then(data => {
    // Process groomed trails
    const groomed = data.GroomingAreas.flatMap(area =>
      area.Trails.filter(t => t.IsGroomed)
    );
    displayOnMatrix(groomed);
  });
Note: Replace {username} and {repo} with your GitHub username and repository name. Enable GitHub Pages in your repository settings to serve these files.

Update Schedule

Data is automatically scraped daily at 7:00 AM MST via GitHub Actions.

Historical data is retained indefinitely. Season typically runs through May 1st.
