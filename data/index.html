Data Structure

Grooming Data

Each resort's grooming data file contains:

  • Date: ISO 8601 timestamp
  • ResortId: Numeric resort identifier
  • GroomingAreas: Array of areas with trail information
    • Trail name, difficulty, open status, groomed status
    • Trail length, type, and other metadata
  • Lifts: Array of lift information (status, wait times, etc.)

Snow Report Data

Each resort's snow report file contains:

  • resort: Resort key (e.g., "vail")
  • date: Date in YYYY-MM-DD format
  • timestamp: ISO 8601 timestamp
  • conditions: Overall snow conditions (e.g., "Early Season", "Powder")
  • snowfall: Snowfall measurements in inches and cm
    • overnight, 24hour, 48hour, 7day, season_total
  • baseDepth: Base depth in inches and cm
  • forecast: Weather forecast data
    • Multiple locations (base, mid-mountain, summit)
    • Temperature hi/low in Fahrenheit and Celsius
    • Weather descriptions, wind, precipitation
    • 5-day forecast

Example Usage

JavaScript / Node.js

// Fetch latest data for all resorts
const response = await fetch('https://jacobschulman.github.io/ski-run-scraper/data/latest.json');
const data = await response.json();

// Access specific resort
const keystoneData = data.keystone.data;
console.log(`Keystone has ${keystoneData.GroomingAreas.length} areas`);

Python

import requests

# Fetch specific resort data
url = 'https://jacobschulman.github.io/ski-run-scraper/data/keystone/2025-01-05.json'
response = requests.get(url)
data = response.json()

print(f"Resort ID: {data['ResortId']}")
print(f"Total areas: {len(data['GroomingAreas'])}")

LED Matrix / IoT

// Fetch only the data you need for a specific resort
fetch('https://jacobschulman.github.io/ski-run-scraper/data/vail/2025-01-05.json')
  .then(res => res.json())
  .then(data => {
    // Process groomed trails
    const groomed = data.GroomingAreas.flatMap(area =>
      area.Trails.filter(t => t.IsGroomed)
    );
    displayOnMatrix(groomed);
  });

Update Schedule

Data is automatically scraped daily at 7:00 AM MST via GitHub Actions.

Historical data is retained indefinitely. Season typically runs through May 1st.
